Ruby Identifiers

Notes and take aways while reading The Well-Grounded Rubyist by David A. Black.

The Identifier Family Tree

Variables

Local x, string, __abc__, start_value, and firstName

  • Local variables start with a lowercase letter or an underscore and consist of letters, underscores, and/ or digits.

Instance @age, @last_name

  • Instance variables serve the purpose of storing information for individual objects. Instance variables always start with a single at-sign and consist thereafter of the same character set as local variables.

Class @@running_total

  • Class variables store information per class hierarchy and follow the same rules as instance variables, ecept that they start with two at-signs.

Global $population

  • Global variables are recognizable by their leading dollar sign. The segment after the dollar sign doesn't follow local-variable naming conventions; there are global variables called $:, $1, and $/, as well as $stdin and $LOAD_PATH. Non-alphanumeric global variables are usually predefined, so you don't need to worry about which punctuation marks are legal.

Constants

Constants A, String, FirstName, and STDIN

  • Constants begin with an uppercase letter. It's customary to use either camelcase or underscore-separated all-uppercase-words in composing constant names from multiple words.

Keywords

class For Class Definitions

def For Method Definitions

if Conditional Execution

__FILE__ The Name of the File Currently Being Execute

  • Ruby has numerous keywords, about 40 of them, and they're generally short, single-word (as opposed to underscore-composed) indentifiers.

Method Names

Method Names x?, string!, __abc__, start_value, and firstName

  • Names of methods in Ruby follow the same rules and conventions as local variables (except that they can end with ?, !, or =). This is by design, methods don't call attention to themselves as methods but rather blend into the texture of a program as, simply, expressions that provide a value. In some contexts you can't tell, just by looking at an expression, whether you're seeing a local variable or a method name-and that's intentional.

Language Semantics

Method Calls, Messages, and Ruby Objects

  • Objects Ruby sees all data structures and values including integers and strings, and also including complex data structures (like arrays) as objects. Every object is capable of understanding a certain set of messages. Each message that an object understands corresponds directly to a method.
  • Method a named, executable routine whose execution the object has the ability to trigger.
  • Object Representation Objects are represented either by literal constructors-like "quotation marks" for strings, or by variables to which they have been bound.
  • Message Sending is achieved via the special dot operator (the message to the right fo the dot is sent to the object on the left of the dot).
